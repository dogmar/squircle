Full Design: Guided Tour + Explorer

Complete architecture for the slideshow system

1. Component Architecture

Component Tree
GuidedExplorer — replaces MathExplorer in index.astro
├─ ExplorerGraphic — extracted SVG viz (left/top)
│  Uses motion.* for animated SVG paths
├─ SlideContent — text panel (right/bottom)
│  Contains <Animate> elements per slide def
├─ SlideControls — Back / Next buttons + step indicator
└─ FreeExplorer — sliders + toggles (final state)

2. Data Types

type GraphicState = {
  showRounded: boolean;
  showSuperellipse: boolean;
  showCorrected: boolean;
  amount: number;
  showMeasurement: boolean;
  measureArc?: "rounded" | "superellipse" | "corrected";
};

type Slide = {
  // React content — uses <Animate> for reveal control
  content: React.ReactNode;

  // Graphic state at each step (partial merge into running state)
  steps: {
    graphic?: Partial<GraphicState>;
    transition?: { duration?: number; ease?: string };
  }[];
};

type AnimateProps = {
  inStep: number;
  outStep?: number;
  children: React.ReactNode;
  initial?: MotionProps; // default: { opacity: 0, y: 8 }
  animate?: MotionProps; // default: { opacity: 1, y: 0 }
  exit?: MotionProps; // default: { opacity: 0, y: -8 }
};

3. State Management

Navigation State

slideIndex: number
stepIndex: number
mode: "tour" | "explore"

Computed Graphic State

Replay all slides[0..slideIndex].steps[0..stepIndex] from initial state, merging each step's graphic partial. Result drives SVG rendering.

Step Context

useSlideStep() hook via React context. Returns current stepIndex within the active slide. <Animate> components read this.

Transition to Free Explore

After last step of last slide, "Next" sets mode: "explore". Sliders/toggles appear. Graphic state becomes user-controlled.

4. ExplorerGraphic (SVG Component)

Extracted from current MathExplorer. Receives GraphicState as props. All SVG elements use motion.path, motion.line, motion.g for smooth transitions:

  • Paths: motion.path with animated d attribute (Framer Motion interpolates SVG path data)
  • Visibility: motion.g with animated opacity (0 or 1) for show/hide
  • Measurement lines: motion.line with animated coordinates
  • Amount changes: Since amount drives the useMemo computation, we animate it as a number using Framer Motion's useMotionValue + useTransform
